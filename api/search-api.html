Introduction

This API provides multiple read-only operations on the TED website data. The endpoint 'Notice Search API' can be used to search notices both TED schema notices and eForms notices.

Important Links

  • TED - Website

  • Expert Search

  • Contact

  • Legal notice

The API is accessible to the general public and can be utilized by anyone in accordance with the terms of usage.

Base URL

https://api.ted.europa.eu/

API Version

Current version: v3

Public API Search Modes

Pagination Mode

The pagination mode is the default mode of the public search API.

This mode is stateless: the user can retrieve any result page, and not necessarily in order. It also means that there is no mechanism to ensure consistency between two retrieved pages: if the user retrieves multiple pages and an OJS is released between two retrieved pages, then the user might miss some notices or have duplicated notices.

Table 1. Required Parameters
Parameter Description Example

limit

The number of notices returned per page

limit=10

page

The page number to retrieve

page=1

Limits for Pagination Mode

  • Maximum number of retrievable notices: 15k

  • Maximum number of notices per page: 250

  • Maximum number of fields per page: 10k

Scroll Mode

The scroll mode allows users to retrieve all notices for a given query without the limitations defined in the pagination mode.

Table 2. Required Parameters
Parameter Description Example

paginationMode

Must be set to "ITERATION"

paginationMode=ITERATION

limit

The number of notices to return for the next page

limit=250

iterationNextToken

Token from previous response (not needed for first request)

token123…​

Limits for Scroll Mode

  • Maximum number of notices per page: 250

  • Maximum number of fields per page: 10k

  • No limit on total retrievable notices

Endpoints

Search Notices

Method

POST

Path

/v3/notices/search

Content-Type

application/json
Request Body Schema
{
  "query": "string",
  "fields": ["string"],
  "page": integer,
  "limit": integer,
  "scope": "LATEST" | "ACTIVE" | "ALL",
  "checkQuerySyntax": boolean,
  "paginationMode": "PAGE_NUMBER" | "ITERATION",
  "onlyLatestVersions": boolean,
  "iterationNextToken": "string"
}

Response Objects

Success Response (200 OK)
{
  "notices": [
    {
      // Notice fields based on requested fields
    }
  ],
  "totalNoticeCount": integer,
  "iterationNextToken": "string"
}
Error Responses
400 Bad Request

Possible error types:

  • QUERY_SYNTAX_ERROR

  • QUERY_INVALID_FIELD_FORMAT

  • QUERY_UNKNOWN_FIELD

  • QUERY_UNSUPPORTED_FIELD_OPERATION

  • QUERY_UNSUPPORTED_FIELD_VALUE

  • UNSUPPORTED_VALUE

  • EXPIRED_ITERATION_TOKEN

  • INVALID_ITERATION_TOKEN

500 Internal Server Error

Returns an error message and additional error details.

FAQ

How can I create, test or validate an expert query?

Use the TED Expert Search page. It provides capabilities to:

  • Write queries

  • Check syntax

  • List all available fields

  • Execute test queries

The validated query can then be used in the API’s query field parameter.

What should I do if the API returns an error about an invalid field, format, or value?

  1. Check the field name against the available fields list

  2. Verify the format matches the field’s requirements

  3. Ensure the value is within the allowed range/format

  4. Test the query on the Expert Search page first
